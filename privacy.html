Privacy Protocol

Effective Date: July 20, 2026

The Dealkly system is structured to strictly respect user privacy and data security. This document outlines how operational data is handled, how system risks are minimized, and how absolute control remains with the user.

1. Information Collection

Only the exact information required to deliver automated retail alerts is processed. This limited data includes:

  • Logistics Data: Target email addresses used solely as alert destinations.
  • Target Parameters: Public retail URLs and price thresholds required for parsing.

Primary account master passwords are never requested, accessed, or stored. All system authentication relies exclusively on isolated, single-purpose application tokens (App Passwords). This guarantees that core login credentials remain invisible to the platform.

2. Data Storage and Security

Sensitive operational configurations, routing strings, and authentication keys are secured within encrypted, server-side environments (GitHub Secrets). These credentials are never written into public repositories, eliminating exposure risks.

The automation pipeline executes within sandboxed, serverless environments. No information is logged, tracked, or cached on secondary hardware, borrowed verification devices, or temporary networks. At the end of each execution cycle, the runtime memory is permanently wiped.

3. Tracking and Third-Party Sharing

No commercial data brokers, persistent background cookies, or tracking pixels are utilized by Dealkly. User parameters are never monetized, traded, or transferred to external marketing entities.

4. User Rights and Deletion

Every user retains full authority over their tracking settings. Removing a tracking parameter or rotating environment security tokens immediately halts all related system activity. This action purges the data from the active system, leaving no residual tracking logs.
